# **视频播放功能增强修改方案**

## **1. 现状分析**
目前项目中的视频预览功能（Preview Pane）通过 [MediaPreview.xaml](file:///d%3A/GitHubProjects/Files/src/Files.App/UserControls/FilePreviews/MediaPreview.xaml) 实现。
- **技术基础**：使用 WinUI 的 `MediaPlayerElement`。
- **当前限制**：
  - 传输控制栏（Transport Controls）设置为 `IsCompact="True"`，功能受限（仅有简单的播放/暂停和进度条）。
  - 没有倍速切换、缩放调整等完整播放器具备的功能。
  - 预览窗口通常较小，适合快速查看，不适合沉浸式播放。

---

## **2. 方案对比**

### **方案 A：在预览窗格增加独立“播放”页签（推荐）**
在信息面板（Info Pane）的“详细信息”和“预览”按钮旁边，增加一个“播放”按钮。
- **优点**：逻辑清晰，用户可以明确区分“快速预览”和“完整播放”。
- **交互**：点击“播放”后，展示一个功能更全、布局更优化的播放器。

### **方案 B：直接修改并增强现有“预览”功能**
对现有的视频预览界面进行重构，支持在“预览模式”和“完整模式”之间切换。
- **优点**：不需要增加新的 UI 元素，保持界面简洁。
- **交互**：增加一个“全屏”或“展开控制栏”的按钮。

---

## **3. 详细修改步骤（以方案 A 为例）**

### **步骤 1：扩展数据模型与枚举**
- **文件**：`InfoPaneTabs.cs` (枚举定义)
- **操作**：增加 `Play` 枚举成员。
  ```csharp
  public enum InfoPaneTabs { Details, Preview, Play }
  ```

### **步骤 2：修改 UI 布局**
- **文件**：[InfoPane.xaml](file:///d%3A/GitHubProjects/Files/src/Files.App/UserControls/Pane/InfoPane.xaml)
- **操作**：在 `StackPanel` 中增加一个新的 `RadioButton`，并根据文件类型（是否为视频/音频）控制其可见性。
  ```xml
  <!-- 播放页签 -->
  <RadioButton
      Content="播放"
      Visibility="{x:Bind ViewModel.IsMediaFile, Mode=OneWay}"
      IsChecked="{x:Bind ViewModel.SelectedTab, Converter={StaticResource EnumToBoolConverter}, ConverterParameter=Play, Mode=OneWay}" />
  ```

### **步骤 3：增强播放器控件**
- **文件**：[MediaPreview.xaml](file:///d%3A/GitHubProjects/Files/src/Files.App/UserControls/FilePreviews/MediaPreview.xaml)
- **操作**：
  1. 增加一个属性 `IsFullMode`。
  2. 根据模式动态调整 `MediaTransportControls` 的属性：
     - 取消 `IsCompact="True"`。
     - 设置 `IsPlaybackRateButtonVisible="True"`。
     - 设置 `IsZoomButtonVisible="True"`。
     - 增加音量条和全屏切换按钮。

### **步骤 4：ViewModel 逻辑适配**
- **文件**：[InfoPaneViewModel.cs](file:///d%3A/GitHubProjects/Files/src/Files.App/ViewModels/UserControls/InfoPaneViewModel.cs)
- **操作**：
  1. 增加 `IsMediaFile` 属性，用于前端绑定按钮可见性。
  2. 在 `UpdateSelectedItemPreviewAsync` 中增加对 `InfoPaneTabs.Play` 的逻辑处理。
  3. 当切换到“播放”页签时，自动开始播放并加载完整控制栏。

---

## **4. 方案优势**
1. **零外部依赖**：完全基于 Windows 内置的媒体库，无需引入第三方播放器。
2. **操作便捷**：用户无需双击打开外部窗口，在右侧面板即可完成完整的视频观看体验。
3. **沉浸感**：通过优化控制栏和可能的“弹出全屏”支持，提供更专业的播放体验。
