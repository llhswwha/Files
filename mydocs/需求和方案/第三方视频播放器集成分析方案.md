# 第三方视频播放器集成分析方案

根据您的需求，在“Files”项目的视频预览小窗口中增加“独立播放”功能，并弹出独立的播放窗口。针对当前项目中的两个插件项目：**LLPlayer** (Plugins\LLPlayer) 和 **mpv.net** (Plugins\mpv.net)，以下是详细的分析对比及集成建议。

---

## 1. 项目概览与对比

| 特性 | LLPlayer | mpv.net |
| :--- | :--- | :--- |
| **核心引擎** | FFmpeg (通过 `FlyleafLib` C# 封装) | `libmpv` (通过 `MpvNet` C# 封装) |
| **开发语言/框架** | C# / .NET 8 / WPF (Material Design) | C# / .NET / WinForms + WPF |
| **主要优势** | **AI 赋能**：集成 ASR (语音转文字)、OCR (视频文字识别) 和多种翻译服务，专为语言学习设计。 | **成熟稳健**：基于工业级的 `mpv` 引擎，支持海量格式，性能极佳且高度可配置（支持脚本）。 |
| **技术架构** | 模块化：`FlyleafLib` (引擎) + `LLPlayer` (UI)。纯 C# 开发，更易于 .NET 项目深度集成。 | 混合式：WinForms 容器 + WPF 弹窗。依赖 `libmpv` 动态库。 |
| **集成难度** | 较低。其核心库 `FlyleafLib` 是纯 C# 开发，便于在 WinUI 3 中调用或启动。 | 中等。需要管理 `libmpv` 的原生 DLL 依赖。 |

---

## 2. 需求场景适配分析

您的需求是：**“点击一个独立播放的按钮，就弹出独立的播放窗口播放视频”**。

### **LLPlayer 的优势**
- **AI 联动**：考虑到您的项目包含 `AI日志` 目录，`LLPlayer` 的 AI 字幕识别和翻译功能与您的项目调性更契合。
- **现代架构**：基于 .NET 8 和 WPF，与 “Files” 项目的技术栈更接近，未来如果需要进行进程内嵌入（In-process Embedding），难度更小。

### **mpv.net 的优势**
- **全能播放**：如果您的首要目标是“万能播放”和“极致性能”，`mpv.net` 是更保险的选择。它对 HDR、高码率视频的支持通常优于自定义封装的 FFmpeg 库。

---

## 3. 推荐集成方案：LLPlayer (针对 AI 与现代架构)

考虑到 `LLPlayer` 已经作为插件存在于您的项目中，且具备强大的 AI 扩展性，**推荐优先考虑集成 LLPlayer**。

### **集成步骤 (修改方案)**

#### **第一阶段：实现“独立窗口播放” (快速实现)**
这是最直接的方案，即在“Files”中点击按钮，以独立进程方式启动播放器。

1.  **修改 `MediaPreview.xaml`**:
    - 在现有的播放控制栏或预览区域上方增加一个“独立播放”按钮（图标可使用 `OpenInNewWindow`）。
2.  **实现 `MediaPreviewViewModel.cs` 中的命令**:
    - 编写一个 `OpenInIndependentPlayerCommand`。
    - 逻辑：获取当前文件的路径，调用 `Process.Start` 启动 `LLPlayer.exe`，并将文件路径作为命令行参数传递。
3.  **适配 `LLPlayer` 启动参数**:
    - 确保 `LLPlayer` 在启动时能正确解析并播放传入的文件路径。

#### **第二阶段：深度集成与状态同步 (进阶)**
如果希望实现更高级的功能（如：点击按钮后，预览窗口停止，独立窗口从当前进度开始播放）：

1.  **引用 `FlyleafLib`**:
    - 在 “Files” 项目中引用 `FlyleafLib` 核心库。
2.  **创建独立播放窗口**:
    - 在 “Files” (WinUI 3) 中创建一个新的顶级窗口，或者封装一个 WPF 窗口容器。
3.  **状态传递**:
    - 启动独立窗口时，将 `MediaPreview` 中当前的 `Position` (进度) 传递给新窗口。

---

## 4. 结论与建议

- **如果您希望播放器具备 AI 辅助学习功能（如翻译、OCR）**：请选择 **LLPlayer**。它是目前项目结构中更具前瞻性的选择。
- **如果您只想要一个稳定、万能的视频播放工具**：请选择 **mpv.net**。

**建议步骤**：
1.  先尝试手动运行 `Plugins\LLPlayer\LLPlayer\bin\Debug\LLPlayer.exe [视频路径]`，确认其播放表现。
2.  在 `Files` 的预览界面添加按钮，通过命令行参数实现最基础的“独立播放”功能。
3.  后续根据使用反馈，决定是否需要将 `FlyleafLib` 深度嵌入到 `Files` 的预览窗格中。
