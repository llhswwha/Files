# 批量视频压缩处理方案 (720P)

根据您的需求，需要实现一个能够批量将文件夹中所有视频压缩为 720P 的功能。考虑到项目中已经集成了 `VideoEditor` 插件（基于 FFmpeg），我们可以利用现有的技术栈来实现这一功能。

---

## 1. 技术选型：FFmpeg

FFmpeg 是目前最强大、最流行的开源多媒体处理工具。由于 `VideoEditor` 插件已经内置了 FFmpeg 的调用逻辑（参见 `FfmpegBatchProcessor.cs`），我们建议直接复用或扩展该逻辑。

---

## 2. 核心处理逻辑

要将视频压缩为 720P（1280x720），典型的 FFmpeg 命令如下：

```bash
ffmpeg -i input.mp4 -vf "scale=-2:720" -c:v libx264 -crf 23 -preset medium -c:a copy output.mp4
```

### **关键参数说明**：
- `-vf "scale=-2:720"`：自动缩放宽度以保持纵横比，高度固定为 720 像素。
- `-c:v libx264`：使用 H.264 视频编码器（通用性最好）。
- `-crf 23`：恒定速率因子（CRF），数值越小质量越高，23 是平衡点。
- `-preset medium`：编码速度预设，平衡压缩率和时间。
- `-c:a copy`：直接复制音频流，无需重新编码（节省时间并保持原音质）。

## 3. 实现方案建议

### **方案 A：扩展 VideoEditor 插件 (推荐)**
在 `VideoEditor` 中增加一个“批量转换”模式。

1.  **UI 调整**：
    - 在 `VideoEditor` 主界面增加一个“批量处理”页签或按钮。
    - 用户可以选择一个文件夹作为输入源，另一个文件夹作为输出目录。
2.  **后端实现**：
    - 利用 `FfmpegBatchProcessor.cs` 管理任务队列。
    - 使用并行处理（Parallel.ForEach）或异步任务（Task）来同时处理多个文件（建议限制并发数以防 CPU 过载）。
3.  **预设支持**：
    - 提供“压缩至 720P”、“压缩至 1080P”等常用预设。

### **方案 B：在 Files 应用中集成简易脚本触发**
如果您希望直接在 `Files` 的文件夹右键菜单中操作。

1.  **自定义操作 (Custom Action)**：
    - 在 `Files.App` 中注册一个右键菜单项“批量压缩至 720P”。
2.  **脚本调用**：
    - 触发后，后台调用一个 PowerShell 脚本或内置的小工具，遍历当前文件夹并调用 FFmpeg 进行处理。
    - 显示一个进度条对话框告知用户当前进度。
